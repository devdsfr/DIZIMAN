Resumo: O DIZIMAN coleta apenas os dados necessários para o funcionamento do sistema de gestão da igreja. Não coletamos CPF, RG ou dados financeiros bancários. Você tem o direito de acessar, corrigir e excluir seus dados a qualquer momento.

1. Quem somos

O DIZIMAN é um sistema de gestão eclesiástica desenvolvido para facilitar o controle de membros, dízimos e ofertas de igrejas. Este documento descreve como tratamos os dados pessoais coletados em nossa plataforma, em conformidade com a Lei Geral de Proteção de Dados (LGPD — Lei nº 13.709/2018).

2. Quais dados coletamos

Coletamos apenas os dados estritamente necessários para o funcionamento do sistema:

  • Dados cadastrais dos membros: nome completo, data de nascimento, telefone, endereço (logradouro, CEP, cidade, estado), e-mail e perfil (membro, líder, pastor).
  • Dados financeiros eclesiásticos: valor e data dos dízimos e ofertas registrados, e o projeto/destinação da oferta. Esses dados são de natureza eclesiástica e não incluem dados bancários, CPF ou RG.
  • Dados de acesso: nome de usuário utilizado para login no sistema.

Não coletamos CPF, RG, dados de cartão de crédito, dados bancários, localização em tempo real ou qualquer dado sensível conforme definido pelo artigo 5º, inciso II da LGPD.

3. Por que coletamos esses dados (base legal)

O tratamento dos dados pessoais é realizado com base nas seguintes hipóteses legais previstas na LGPD:

  • Execução de contrato/legítimo interesse: para permitir o gerenciamento dos membros da igreja e o registro de contribuições eclesiásticas.
  • Obrigação legal: registros financeiros de organizações religiosas podem ser exigidos para fins contábeis e comprovação junto a órgãos reguladores.
4. Como utilizamos os dados

Os dados coletados são utilizados exclusivamente para:

  • Cadastro e gestão dos membros da igreja;
  • Registro e controle de dízimos e ofertas;
  • Geração de relatórios financeiros para uso interno da igreja;
  • Autenticação e controle de acesso ao sistema.

Os dados não são compartilhados com terceiros, vendidos, alugados ou utilizados para fins publicitários.

5. Armazenamento e segurança

Os dados são armazenados em banco de dados PostgreSQL hospedado na plataforma Render.com, localizada em servidores nos Estados Unidos, com acesso restrito por credenciais seguras. A comunicação entre o aplicativo e o servidor utiliza protocolo HTTPS/TLS.

Aplicamos medidas técnicas e organizacionais adequadas para proteger os dados contra acesso não autorizado, perda, alteração ou destruição.

6. Seus direitos (LGPD)

Como titular dos dados, você tem os seguintes direitos garantidos pela LGPD:

  • Acesso: solicitar confirmação e acesso aos seus dados;
  • Correção: solicitar a atualização de dados incompletos ou incorretos;
  • Exclusão: solicitar a eliminação dos seus dados;
  • Portabilidade: solicitar a exportação dos seus dados;
  • Revogação do consentimento: retirar o consentimento a qualquer momento;
  • Oposição: opor-se ao tratamento em determinadas circunstâncias.

Para exercer seus direitos, entre em contato com o administrador do sistema da sua igreja.

7. Retenção dos dados

Os dados são mantidos enquanto o membro estiver ativo no sistema ou enquanto necessário para cumprimento de obrigação legal. Após solicitação de exclusão, os dados serão removidos em até 30 dias, salvo obrigação legal de retenção.

8. Encarregado de Dados (DPO)

O responsável pelo tratamento de dados é o administrador da instalação do DIZIMAN em cada igreja. Dúvidas sobre o tratamento de dados devem ser direcionadas ao administrador local ou ao desenvolvedor pelo e-mail: devdsfr@gmail.com.

9. Alterações nesta política

Esta política pode ser atualizada periodicamente. Alterações significativas serão comunicadas por meio do sistema. O uso continuado do sistema após as alterações implica aceitação da nova versão.

Aviso de Cookies e Armazenamento Local

LGPD

Última atualização: {{ today | date:'dd/MM/yyyy' }}

Resumo: O DIZIMAN utiliza apenas armazenamento local (localStorage) para manter sua sessão ativa. Não utilizamos cookies de rastreamento, publicidade ou analytics de terceiros.

1. O que é o armazenamento local?

O armazenamento local (localStorage) é uma tecnologia do navegador que permite que aplicações web guardem pequenas quantidades de informação no dispositivo do usuário. Diferentemente dos cookies tradicionais, o localStorage não é enviado automaticamente ao servidor a cada requisição.

2. O que o DIZIMAN armazena no seu dispositivo

O DIZIMAN utiliza o localStorage do navegador exclusivamente para:

  • Sessão de autenticação (chave: diziman_user): armazena o nome de usuário e o perfil de acesso (ex: administrador, pastor, líder) para manter você logado enquanto usa o sistema. Nenhuma senha é armazenada em texto claro.

Não utilizamos:

  • Cookies de rastreamento;
  • Cookies de publicidade ou retargeting;
  • Ferramentas de analytics de terceiros (Google Analytics, Meta Pixel, etc.);
  • Cookies de sessão persistentes além dos necessários para autenticação.
3. Por que usamos o armazenamento local

O armazenamento da sessão é estritamente necessário para o funcionamento do sistema. Sem ele, você precisaria fazer login a cada página acessada. Esta é a base legal de interesse legítimo e necessidade técnica previstas na LGPD.

4. Como limpar os dados armazenados

Você pode remover os dados armazenados pelo DIZIMAN a qualquer momento:

  • Pelo sistema: clique em "Sair" (logout) no menu principal. Isso remove automaticamente os dados da sessão.
  • Pelo navegador: acesse as configurações do seu navegador → Privacidade → Limpar dados de navegação → Dados de sites. Ou pressione F12 → Application → Local Storage → clique com botão direito e limpe os dados do domínio.
  • No Android (app): acesse Configurações do dispositivo → Apps → DIZIMAN → Armazenamento → Limpar dados.
5. Dados de terceiros

O DIZIMAN utiliza os seguintes recursos de terceiros que podem ter suas próprias políticas:

  • Bootstrap (via CDN jsDelivr): biblioteca de interface visual. O jsDelivr pode registrar acessos em seus logs de servidor, mas não rastreia usuários individualmente.
  • Bootstrap Icons (via CDN jsDelivr): mesma política acima.
  • Render.com: plataforma de hospedagem do sistema. Pode registrar logs de acesso padrão (IP, timestamp, endpoint). Consulte a política de privacidade da Render.com para mais detalhes.
6. Suas escolhas

Você pode configurar seu navegador para bloquear o armazenamento local. No entanto, isso impedirá o funcionamento correto do sistema, pois a autenticação não poderá ser mantida entre as páginas.

Ao utilizar o DIZIMAN, você consente com o uso do armazenamento local conforme descrito neste aviso.
